# Пояснение к схемам событий

- `schemas/events/*.yaml` описывают минимальные поля для журналирования в контексте финмониторинга (AML/KYC, авторизация, транзакции).
- `schemas/pipelines/*.yaml` дают ориентир по нормализации полей для логоприемников (ELK/Loki).
- Поля `event_type` и `ts` обязательны для всех событий; остальные поля подбираются по кейсу.
- Значения из `use_cases` помогают настроить маршрутизацию и дашборды (пример: fraud_detection, aml, audit).
