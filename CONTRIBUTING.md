# Contributing

Спасибо за участие! В этом проекте действуют правила: **сначала тесты, потом код**.

1. Перед изменениями опишите/дополните юнит-тесты (валидаторы схем, CLI, AI stubs, санкционные клиенты).
2. Затем реализуйте код под эти тесты.
3. Перед любым PR/коммитом обязательно запускайте единый скрипт:

```bash
./scripts/dev_run_all_tests.sh
```

Он включает линтеры и `pytest`. Интеграционные тесты можно запускать отдельно при наличии инфраструктуры.

Прочее:
- не добавляйте реальные ключи, ПДн, конфиги прод-систем;
- уважайте лицензии и регуляторные ограничения;
- используйте английский/русский комментарии там, где код сложен.
